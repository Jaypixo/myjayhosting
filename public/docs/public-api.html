The public files API.

Ask, from any code running anywhere, exactly what files a published site has. No scraping required.

The problem this solves

Outside of MyJay itself, there was no way to find out what's on a published site short of fetching its rendered pages and parsing whatever links happen to be in them, which misses anything not linked from somewhere. This is a direct, structured answer instead: give it a username, get back every file that site actually has.

The endpoint

GET /api/sites/:username/files, where :username is the subdomain itself, the part before .myjay.net, lowercase, no protocol.

$ curl https://myjay.net/api/sites/noah/files {"username":"noah","files":[ {"key":"index.html","size":1653,"modified":"2026-06-21T13:13:02.355Z"}, {"key":"styles.css","size":4448,"modified":"2026-06-21T13:13:02.641Z"}, {"key":"blog/post-1.html","size":6001,"modified":"2026-06-21T13:13:02.846Z"} ]}

What you get back

FieldTypeMeaning
usernamestringThe username you asked about
filesarrayEvery file on the site, see below
files[].keystringPath relative to the site root, e.g. blog/post-1.html
files[].sizenumberSize in bytes
files[].modifiedstringISO 8601 timestamp of when that file was last uploaded

Order isn't guaranteed, and folders aren't a separate concept here, same as everywhere else on the platform, see file manager. A key with a slash in it is just a file whose path happens to contain one.

Only published sites are listed

This follows the exact same two-step check a browser hitting username.myjay.net goes through, see how routing works for the full version:

SituationStatusBody
Username was never registered404"No site is registered at this subdomain"
Registered, but still in draft403"This site exists but has not been published"
Registered and published200the file listing

This isn't extra exposure beyond what already exists: it's the same not-claimed-versus-not-published distinction anyone gets by just visiting the subdomain directly, only as a status code instead of a page. A draft site's contents are never listed, regardless of who's asking.

No key, no session, any origin

Every other endpoint on the platform either needs you to be logged in, or only accepts requests from myjay.net itself. This one doesn't: no API key, no cookie, and it answers requests from any origin, specifically so it's usable from a browser tab on a completely different site, a build script, a bot, whatever's asking. There's nothing to sign up for.

What's not included

  • File contents. This is a listing, not a download. To read a file, request it directly: https://noah.myjay.net/blog/post-1.html works exactly like it would for a visitor, see how routing works.
  • Empty-folder placeholders. Folders with nothing real in them don't produce an entry, there's nothing to list.
  • Anything about draft sites, including whether the username is registered at all versus just unpublished, that distinction only resolves once it's actually live, see the table above.
